You can also enable tab functionality through javascript if you do not use the data-toggle="tab" markup. Each tab would need to be activated individually.

    $('#myTabs a').click(function (e) {
      e.preventDefault()
      $(this).tab('show')
    })

You can activate individual tabs in several ways:

      $('#myTabs a[href="#profile"]').tab('show') // Select tab by name
      $('#myTabs a:first').tab('show') // Select first tab
      $('#myTabs a:last').tab('show') // Select last tab
      $('#myTabs li:eq(2) a').tab('show') // Select third tab (0-indexed)

To make tabs fade in, add .fade to each .tab-pane. The first tab pane must also have .in to make the initial content visible.

    <div class="tab-content">
      <div role="tabpanel" class="tab-pane fade in active" id="home">...</div>
      <div role="tabpanel" class="tab-pane fade" id="profile">...</div>
      <div role="tabpanel" class="tab-pane fade" id="messages">...</div>
      <div role="tabpanel" class="tab-pane fade" id="settings">...</div>
    </div>

For more tab methods and events please refer to the bootstrap documentation at http://getbootstrap.com/javascript/#tabs-methods

Input groups

Extend form controls by adding text or buttons before, after, or on both sides of any text-based <input>. Use .input-group with an .input-group-addon or .input-group-btn to prepend or append elements to a single .form-control.

Textual <input>s only

Avoid using <select> elements here as they cannot be fully styled in WebKit browsers.

Avoid using <textarea> elements here as their rows attribute will not be respected in some cases.

Tooltips & popovers in input groups require special setting

When using tooltips or popovers on elements within an .input-group, you'll have to specify the option container: 'body' to avoid unwanted side effects (such as the element growing wider and/or losing its rounded corners when the tooltip or popover is triggered).

Don't mix with other components

Do not mix form groups or grid column classes directly with input groups. Instead, nest the input group inside of the form group or grid-related element.

Basic example

Place one add-on or button on either side of an input. You may also place one on both sides of an input.

We do not support multiple add-ons (.input-group-addon or .input-group-btn) on a single side.

We do not support multiple form-controls in a single input group.

Accordion

This component uses the jQuery UI widget accordion. The accordion wrapper has an id of #accordion. All headings should h3.

Accordions are styled to be all closed upon initiation, have no animation and expand to height of the content in the panel. For more options on accordions please see documentation at http://api.jqueryui.com/accordion/

If this is for third party site you will need to add the jQuery UI library.
Example:
<script src="//ajax.googleapis.com/ajax/libs/jqueryui/1.10.4/jquery-ui.min.js"></script>
and instantiate the accordion like so:
$('#accordion').accordion();
There are additional options for accordions which can be viewed here: http://api.jqueryui.com/accordion/
